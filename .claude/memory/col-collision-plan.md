Plan to parse RenderWare **COL** collision and bind it to placed objects: `.claude/plans/007-col-collision.md`. Builds on [[map-pipeline]] / [[binary-ipl-render-approach]]; consumed later by Rapier physics (plan 006 phase 6 / future plan 008).

**Verified data facts (checked the archive directly):** the user re-packed `.col` into the **main** `static/models/gta3.img` (WIMG, now ≈758 MB; `.dff` 12217, `.txd` 2163, `.col` 215). The 215 `.col` entries are **collision libraries** = **8061** collision models total; versions **COL2 (762) + COL3 (7299)**, no COL1/COL4. **100% of collision-model names match a `.dff`** in the same archive → bind by lowercased model name (like `modelKey`/`getClump`). No separate collision archive / second download — the index is built from the already-in-memory `ImgArchive` (`*.col` entries).

**VERIFIED format (Phase 1 done — `src/renderware/parsers/binary/col.ts` `parseColLibrary`):** COL is **not** RW-chunked — a `.col` file is a flat sequence of `FileHeader{ fourcc[4], uint32 size }` blocks, next model at `pos + 8 + size`. Body (offsets from body start): `@0 name[22]` (NUL-terminate at first `\0`), `@22 u16 modelId`, `@24 bounds` (min,max,center,radius = 40B), `@64 u16 numSpheres, u16 numBoxes, u32 numFaces, u32 flags, u32 off{Spheres,Boxes,Cones,Vertices,Faces}`; COL3/4 add `numShadowFaces + 2 shadow offsets`. **Offset base = body start − 4** (`bodyPos = stored − 4`) — this was the #1 trap. **Vertices are compressed int16/128 (6B) in BOTH COL2 and COL3** here (corrected: float verts are COL1 only, absent); count not stored → `numVertices = max(face index)+1`. Faces `u16 a,b,c + u8 material + u8 light` (8B). Spheres 20B (center,radius,surface), boxes 28B (min,max,surface); surface = `u8 material,flag,brightness,light`. Data model `ColModel`: bounds + spheres + boxes + trimesh (`vertices: Float32Array` flat n*3, `faces: ColFace[]`). Added `BinaryStream.i16()`.

**Index (Phase 2 DONE — `src/renderware/collision/collision-index.ts`):** `buildCollisionIndex(archive)` walks every `*.col` entry in the in-memory `ImgArchive` → `CollisionIndex = Map<lowercased name, ColModel>` (cached per archive via `WeakMap`, first-name-wins, parse failures skipped); `getCollision(index, name)` for binding. Exported via `collision/index.ts` + `renderware` barrel.

**Binding (Phase 3 DONE — `src/renderware/collision/build-colliders.ts`):** `buildColliders(index, defs, {center, radius})` mirrors `buildRegion` — filter (`interior===0`, radius, **non-LOD**), group by model name, `getCollision`, emit `RegionColliders[]` = `{ col, name, transforms: Matrix4[] }`. Transform = position + **conjugated** IPL quat, unit scale, **no per-part frame matrix** (COL verts already model-space). Coords stay GTA Z-up.

**Debug viz (Phase 4 DONE):** `renderware/three/build-col-wireframe.ts` `buildCollisionWireframe(colliders)` → one `LineSegments` (tri edges + box edges + sphere rings, transformed per placement, model-space Z-up). `WorldAdapter.loadCollisionDebug` + `GtaSaWorldAdapter` impl wrap it in the −90°X group. `Config.showCollision`; `Game.setShowCollision`/`refreshCollision` (tracks `collisionObjects` + `lastRequest`, rebuilds on region reload, disposes on toggle-off; `disposeObject` now handles `LineSegments`). Debug overlay has a **COLLISION: Show/Hide** group. `ui/canvas-host` bootstrap config gained `showCollision: false`. Browser acceptance (Ctrl+D → Show on Ganton → green wireframes on buildings) pending user.

**Physics seam (Phase 5 DONE):** generic engine-side DTOs in `game/interfaces/collider.interface.ts` (`ModelColliders` = `{ name, shape: { vertices, indices, boxes, spheres }, transforms: Matrix4[] }`) — no renderware types in `game`. `WorldAdapter.loadColliders` reserved + implemented (`toModelColliders` in the adapter converts `RegionColliders`/`ColModel` → DTO). `game/collision/collision-world.ts` `CollisionWorld` holder (`models`/`placementCount`/`set`/`clear`); `Game.getCollisionWorld()` + `Game.loadColliders()` (explicit — NOT auto in `loadGame`, to avoid parsing all 8061 `.col` on startup; a future `PhysicsSystem` calls it). `loadGame` clears it (region-scoped). Boundary holds.

Remaining: browser visual acceptance of the collision overlay; **Phase 6 = bitECS + Rapier physics — now its own plan `.claude/plans/008-character-physics.md`** (first character spawn, see [[player-cube-placeholder]]).
