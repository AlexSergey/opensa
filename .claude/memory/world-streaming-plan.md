Plan: `.claude/plans/009-world-streaming-lod.md`. Render the map like GTA SA: full-detail (HD) models around the view, **LODs** ringing them, world split into **grid cells (squares)** that stream in/out as the view moves; **configurable draw distance** for HD models and for LODs. This is the **StreamingSystem** scaffolded in plan 006 (rides the `System` registry + player/camera view).

**SA reference:** per-instance `drawDist` (IDE, we parse `IdeObjectDef.drawDistance`) + **LOD links** (`IplInstance.lod` = index of the LOD stand-in instance; LOD models are `lod`-prefixed = `isLodModel`, with large drawDist). HD culled when far → its LOD draws → far = nothing. `CWorld` buckets the world into a sector grid. (Memory streaming N/A — our WIMG archive is fully in RAM.)

**Our design (grid streaming):** bucket exterior (`interior===0`) instances into `cellSize` cells, split HD vs LOD per cell; `buildCell(...)` builds/caches per-cell `InstancedMesh[]` (reuse build-region grouping/conjugated-quat). Each cell-crossing: cells within `hdDrawDistance` → HD, within `lodDrawDistance` (beyond HD) → LOD, else unloaded; StreamingSystem diffs desired vs loaded → `adapter.loadCell`/unload under one −90°X streaming root. Generic `game/streaming/{streaming.system,grid}.ts` (cell/ring math, no renderware) + `renderware/map/{world-grid,build-cell}.ts` behind the adapter (`loadCell(CellRequest)`+`cellSize`). New `Config.streaming` = `{ cellSize, hdDrawDistance, lodDrawDistance }` (defaults ~250/300/1500, tunable). Status: **planned, not started.**

**Debug UX (decided):** entering debug mode **suspends streaming** → StreamingSystem `manual` mode rendering an explicit cell set; debug overlay shows an **N×N checkbox grid** of cells around the current section (centre/current checked by default; tick extra sections to view alongside) + a single **"Show LODs"** toggle (HD vs LOD for checked cells). Leaving debug → resume streaming flow. Replaces the old `geometry: lods|map` + `camera: ganton/full-map` toggles. So StreamingSystem has two modes (`stream`/`manual`, follows `debugMode`) and exposes view cell + cellSize + `setManualCells(cells, lod)`.

**Cell fade-in (DONE, roadmap "Streaming fade objects"):** `game/streaming/fade.ts` `CellFader` ramps a
freshly-streamed cell's material opacity 0→original over 0.4s so sections don't pop. Wired in
`StreamingSystem`: `update(delta=0)` advances fades; on cell add → `fader.start(key, objects)` (skipped
when `config.mapViewer` — inspector shows instantly); on unload → `fader.cancel(key)` restores materials
before the cell mesh returns to the adapter cache. Safe because materials are **per-cell** (built fresh
in `buildClumpParts`, only Textures/clumps are shared) — verified. Translucent materials keep their
original opacity/`transparent`; opaque ones toggle `transparent` only for the fade (with `needsUpdate`),
restored on completion. `fade.test.ts` (5 tests).
