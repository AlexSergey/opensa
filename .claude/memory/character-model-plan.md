Plan: `.claude/plans/011-character-model-tommy.md`. Replace the temp 3ds cube ([[player-cube-placeholder]]) with a real GTA-SA character — **Tommy Vercetti** (`static/player/tommy.dff` + `tommy.txd`, SA-adapted, single mesh). Tommy spawns at `PLAYER_SPAWN` instead of the cube, textured + upright, interacts with the streamed collision exactly like the cube (dynamic box body, plan 008/010), and carries his **skeleton** as a real `THREE.Skeleton`/`SkinnedMesh` in **bind pose** — NO animation in this task (the *animation manager* is the explicit next task).

**Model verified valid:** `parseDff` → 1 atomic, 33 named frames (Root/Pelvis/Spine/Neck/Head/arms/legs/fingers/toes), 1 geometry (1153 v, 1355 tri, flags 0x36, normals yes, 1 UV, material → texture `player`). bbox ≈ ±0.9 in two axes (T-pose; height + arm-span ≈ 1.8) and ±0.18 depth. `tommy.txd`: 1 texture `player` 256×256. It is **skinned** — the SkinPLG (`0x0116`) geometry extension is skipped by our parser today; fine for a static bind-pose render (verts already in bind pose), but must be parsed for the skeleton.

**Key facts / decisions:** layer rule lets `renderware` + `game/adapters/**` + `ui` import the DFF loaders (not `game/**`) — load Tommy via a new `WorldAdapter.loadCharacter(dff,txd)` reusing `DFFLoader`/`TXDLoader`/`buildClump` (the standalone-viewer path, [[prelit-darkness-and-model-viewer]]). Collision stays a **box** sized to a human (~`[0.3,0.3,0.9]`), decoupled from the T-pose mesh bbox, with a foot offset — `setupCharacter` currently auto-sizes from the bbox and must take explicit half-extents. Orientation/scale/offset need **browser tuning** (SA ped bind pose isn't world-Z-up). Iterations: 1) static bind-pose Tommy replaces cube + collision; 2) parse SkinPLG in `dff.ts`; 3) build `THREE.SkinnedMesh`+`Skeleton` (bind pose, expose on `CharacterContext`); 4) retire cube + cleanup. See [[character-physics-plan]] for the existing player wiring.

**Iter 1 DONE (code; browser pending):** `WorldAdapter.loadCharacter(dff,txd)` (GtaSaWorldAdapter: TXDLoader+DFFLoader, `setConvertToYUp(false)` = native model space). New `game/character/orient-character.ts` `orientCharacter(model, {rotation,scale}, boxHalfZ)` → wraps the model in a `Group` (render-sync syncs the wrapper so the per-frame Transform doesn't clobber the stand-up correction on the inner model; centres X/Y, drops feet to box base). `setupCharacter(…, halfExtents?)` takes an explicit human box. Bootstrap: `PLAYER_HALF_EXTENTS=[0.3,0.3,0.9]`, `TOMMY_PLACEMENT={rotation:[π/2,0,0],scale:1}` (bone audit: model up=+Y, side=+X, height≈1.56). Cube no longer loaded (`load-player.ts`/`player.3ds` still on disk, retired in iter 4). 180 tests green. **Browser ✅ — Tommy stands upright on the ground** (placement correct as-is).

**Iter 2 DONE:** parse the Skin plugin. `RwSection.SKIN=0x116` (inside the Geometry's Extension, next to BinMeshPLG); `dff.ts` `parseSkinExtension` → `RWGeometry.skin?: RWSkin {boneIndices(Uint8, nv×4), boneWeights(Float32, nv×4), inverseBindMatrices(Float32, numBones×16), numBones, usedBones[]}`; non-skinned geometries keep `skin=undefined` (map models unchanged). SA layout: `u8 numBones,numUsedBones,maxWeights,pad`, `usedBones[numUsedBones]`, `nv×4 u8` indices, `nv×4 f32` weights, `numBones×16 f32` inverse-bind, then a **12-byte trailer skipped**. Inverse-bind matrix RW layout = `right.xyz,0, up.xyz,0, at.xyz,0, pos.xyz,0` → for iter 3 a `THREE.Matrix4` has columns (right,up,at,pos), same transpose as `build-clump frameMatrix`. Tommy: numBones 32 (vs 33 frames — frame 0 is the root null), weights all sum to 1. 182 tests green.

**Iter 3 DONE (code; browser pending):** `renderware/three/build-skinned-clump.ts` `buildSkinnedClump(clump, textures) → {root, skeleton, bonesByName} | null` — `Bone` per frame (local transform via the `frameMatrix` transpose), parented per `parentIndex`; `SkinnedMesh` with `skinIndex`/`skinWeight` attrs + shared `buildMaterial`/`groupTrianglesByMaterial` (now exported from `build-clump`). **Bone mapping: skin bone i ↔ frame i+1** (verified by matching inverse-bind matrices to frame world matrices; frame 0 = dummy root). Uses `new Skeleton(bones)` (auto inverses) so **bind pose = raw mesh regardless of mapping** — mapping only matters for animation. `WorldAdapter.loadCharacter` now returns `CharacterModel {object, skeleton, bonesByName}` (adapter: fetch buffers → parseDff/parseTxd/buildTextureMap → buildSkinnedClump, fallback buildClump). `setupCharacter(…, {halfExtents, skeleton, bonesByName})` → `CharacterContext` exposes `skeleton`+`bonesByName` for the animation manager. 185 tests green.

**Orientation correction (during plan 012 iter 3):** the iter-1 `TOMMY_PLACEMENT.rotation = [π/2,0,0]` was only needed to stand the static **bind T-pose** (model up = +Y). With animation, the **IFP clips orient the skeleton into GTA Z-up (animated up = model +Z)** — so the model needs **NO rotation** (`rotation:[0,0,0]`); the `+90°X` on top laid Tommy on his back. Also simplified `orientCharacter` to rotation+scale+`offset` only (dropped the bind-bbox foot-drop — wrong for the animated pose; the model origin/pelvis sits at the box centre, `offset:[0,0,-0.07]` nudges feet to the box base). Takeaway: **don't pre-rotate an animated ped; the clip provides GTA-Z-up orientation.**

**Iter 4 DONE — PLAN 011 COMPLETE.** Removed the cube placeholder (`game/character/load-player.ts` + `static/player/player.3ds` deleted; [[player-cube-placeholder]] now historical). Player is the Tommy skinned DFF (bind pose, real `Skeleton`), spawns + lands/walks with streamed collision, camera follows. 185 tests / tsc / eslint / build green. **Next task: the animation manager** — clip playback driving `CharacterContext.skeleton`/`bonesByName` (recall: skin bone i ↔ frame i+1 for animating the right bones).
