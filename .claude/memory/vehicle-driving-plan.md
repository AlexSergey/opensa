Plan: `.claude/plans/017-vehicle-driving.md`. Drive the seated car with WSAD (arcade kinematic, not full Rapier vehicle). Builds on [[enter-vehicle-plan]] (seated state in `EnterVehicleSystem`) + [[vehicle-loading-plan]] (`VehicleRig` wheels) + parsed `handling.cfg` dict.

**Approach:** integrate scalar `speed` + `heading` from input each frame, mutate the car `object` transform (native Z-up, pos + rotation.z); teleport the seated player to the moving seat; `rig.setSpeed/setSteer`; hold `CAR_sit` facing the live heading (`setScripted` each frame — `AnimationController.play` no-ops on same clip). Folded into `EnterVehicleSystem` (owns seated state + keyboard/physics/body/animation/camera); needs `config.controls` + the car `object`/`rig` added to `EnterableVehicle`. **`driveState {pos, heading, speed}`** from `startSeated`; seat + EXIT read it (static `position`/`heading` = parked placement, used only for approach/open). 

**Iter 2 DONE (handling.cfg feel):** adapter `vehicleHandling(handlingId)` reads handling.cfg fields by index — [11] maxVelocity, [12] engineAccel, [16] brakeDecel, [19] steeringLock° — into `VehicleModel.handling` (`VehicleHandling` interface; sane fallbacks 160/22/8.5/30). `EnterableVehicle.handling`; `drive` scales per-car (consts in enter-vehicle.system): `maxSpeed=maxVel·SPEED_SCALE(0.15)`, `accel=engineAccel·ACCEL_SCALE(0.7)`, `drag=·DRAG_SCALE(0.35)`, `brake=brakeDecel·BRAKE_SCALE(2.5)`, `turnRate=steeringLock·TURN_PER_DEG(1.7/30)`, wheel steer = steeringLock·π/180, reverse=max·0.4. `integrateSpeed` now takes accel/brake/drag. Per-car verified. **Differences were too faint (linear scale + close handling values) → widened:** baseline-remap `maxSpeed=max(8,(maxVel−60)·0.25)`, `accel=max(5,(engineAccel−8)·1.3)`, `drag=accel·0.4`, and **turn now mass-scaled** `turnRate=steeringLock·(1.7/30)·(1600/mass)` (added `mass`=field[0] to VehicleHandling). Now camper 15/10.4/1.43, admiral 26/18.2/1.65, infernus 45/28.6/1.94 — clearly distinct. Tune SPEED_BASE/SCALE, ACCEL_BASE/SCALE for balance.

**Iter 3 DONE (collision, browser pending):** car↔world via a **kinematic box collider + character controller** per car. `PhysicsWorld.createKinematicBox(pos, [hx,hy,hz], heading)` + `moveCar(controller, body, collider, desired, heading)` (orients box to heading, `computeColliderMovement` → slide, **keeps z fixed** — arcade flat-ground — returns planar movement). `drive` moves via moveCar (not raw position add); on a wall (moved < ½ desired) it bleeds speed. The kinematic box **replaces the static COL collider** → it's the parked obstacle AND moves with the car (no ghost). Player collider disabled while seated (`setColliderEnabled(playerCollider,false)` on getin, true on exit) so the rider doesn't block the car. halfExtents now `[hx,hy,hz]` from COL bounds (adapter). Enter system regained `physics`+`playerCollider` ctor args; EnterableVehicle += `body`/`collider`/`controller`. **Caveats:** box approximates the body (not trimesh); fixed z (no slopes/terrain follow — later); created per-car controller. Tests green.

**Iterations:** 1) kinematic drive core (move/steer/wheels/player-rides/cam, exit uses live transform, no car↔world collision, simple constants). 2) handling.cfg feel per car. 3) collision (kinematic car body + move the parked COL collider so it's not a ghost). 4) polish (handbrake/reverse/no-exit-at-speed/lean/audio). **Iter 1 DONE (browser pending):** `EnterVehicleSystem.drive(delta)` runs in the `seated` phase. WSAD (`config.controls`, passed as 9th ctor arg): `integrateSpeed` (accel 14 / brake 26 / drag 7, clamp [-9, 24]); heading turn (rate 1.7·(speed/max)·sign). Mutates the car's own `position`/`heading` (live transform) + sets `object.position`/`rotation.z`; teleports player to the moving seat; `setScripted('car_sit',{facing:heading})` each frame (play no-ops same clip → CJ turns with car); `rig.setSpeed(speed)`/`rig.setSteer(steer·0.5)`; `aimCamera(heading)` chase. `EnterableVehicle` += `object`, `rig`; canvas-host passes a **copy** of position (mutated) + `game.getConfig()`. Exit now correct after driving (uses live position/heading). **No car↔world collision (drives through); simple constants.** 10 enter-system tests; all green.

**Jitter fix:** car jittered/wobbled because `drive` (car move + player teleport) ran in `update` (per render frame) while physics ran in `fixedUpdate` (fixed step) — cadence mismatch. Moved driving to **`EnterVehicleSystem.fixedUpdate(step)`** (lockstep with physics, like walking). Also `PhysicsWorld.teleport` now sets **both** `setTranslation` + `setNextKinematicTranslation` (kinematic body otherwise gets pulled back to its stale next-translation each step). If residual 1-tick lag wobble remains: PhysicsSystem reads body→Transform before enter.fixedUpdate (registered later), so player Transform/camera lag the car object by one tick — would need reorder or direct Transform write.

**Camera while driving (final):** user wants the camera **centred once** behind the car on seat, then **free to orbit** (mouse) during driving — NOT a continuous chase. Both the per-tick snap AND the eased auto-center fought the horizontal mouse → wobble + pull-back. So: camera is a single `aimCamera(azimuth)` snap callback again; `startSeated` snaps behind once, `drive` does NOT touch the camera (free orbit), exit snaps behind the player. Auto-center machinery (setAutoAzimuth/wrapAngle/Game.setFollowAutoAzimuth) removed. Driving is camera-independent (W/S/A/D = throttle/steer, not camera-relative), so free camera is fine. All green.

**Coast-to-stop on exit:** Enter while driving fast → `stopping` phase (firm `STOP_BRAKE=18`, no input) until speed ≤ `STOP_THRESHOLD=1` → then the normal exit sequence; near-stationary Enter exits immediately. `drive` runs for `seated`+`stopping`. (Edge-detect needs an update frame with Enter released between enter & exit — real game has them.)

**1-tick lag fix (player drifted back in seat while driving):** `PhysicsSystem` reads body→Transform BEFORE `EnterVehicleSystem.fixedUpdate` (registered later), so the seated player rendered one fixed step behind the car (≈ speed·dt back). Added `CharacterContext.placePlayer(pos)` = `physics.teleport` + **direct ECS `Transform` write** (so render-sync/camera use the current seat this frame). Enter system now takes `placePlayer` (dropped `physics`+`bodyHandle` ctor args) and uses it for getin/drive/exit. All green. **VERIFY/TUNE IN BROWSER:** speeds/turn rate, steer sign, chase-cam feel (aimCamera every frame disables mouse look in-car).
