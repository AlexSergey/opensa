Plan: `.claude/plans/013-player-physics.md`. The player is currently a **dynamic cuboid** with `lockRotations` whose velocity the controller sets directly (`setLinvel`) — it sticks on poles, has no inertia, and can't do steps/slopes. Replace it with a **Rapier `KinematicCharacterController` + vertical capsule**.

**Why this works (verified):** Rapier 0.19 (`@dimforge/rapier3d-compat`) has the full kinematic character controller — `world.createCharacterController(offset)`, `enableAutostep` (steps/kerbs), `enableSnapToGround` (stay glued on slopes/stairs), `setMaxSlopeClimbAngle`, `computeColliderMovement`→`computedMovement()` (slides along obstacles) + `computedGrounded()`; body = `kinematicPositionBased`, moved via `setNextKinematicTranslation` before `world.step()`. `ColliderDesc.capsule(halfHeight, radius)` is **Y-aligned** → rotate +90°X to stand vertical in our **Z-up** world.

**GTA SA note:** the per-bone sphere(head)+capsules/boxes are the **hit/ragdoll** collision (shooting/ragdoll), NOT movement — movement uses a capsule. So a single vertical capsule is correct here; the multi-shape hitbox is a later task.

**Design:** kinematic capsule (radius ≈0.3, total ≈1.8). We own a **`Velocity` ECS component** (the kinematic body reports no `linvel`): each fixed step accelerate horizontal toward the camera-relative input target (ramp-up/decel/airControl = inertia), integrate gravity into `vz` (kinematic ignores world gravity), jump sets `vz`; `computeColliderMovement(capsule, velocity·dt)` → set kinematic translation; write `Velocity`+`grounded` to ECS. `CharacterAnimationSystem` must read speed/grounded from ECS `Velocity`+`grounded` (NOT `physics.getLinvel`/`isGrounded`). Controller config: autostep ~0.4/0.1, snap ~0.4, maxSlope ~50°, offset ~0.02. `MovementConfig` += `accel`/`deceleration`/`airControl` ([[animation-manager-plan]] already put walk/run/jump speeds there).

**Iter 1 DONE:** `PhysicsWorld` plumbing (no wire-in, no behaviour change). `createCharacterController(offset=0.02)` — **`setUp({0,0,1})`** (critical, Z-up), slide, `enableAutostep(0.4,0.1,true)`, `enableSnapToGround(0.4)`, maxSlope 50°/minSlide 45°, `setApplyImpulsesToDynamicBodies(true)`. `createKinematicCapsule(pos,r,halfHeight)` → kinematic body + Z-aligned capsule (Rapier capsule Y-aligned → collider rotation +90°X) → `{body, collider}` handles. `moveCharacter(controller, body, collider, desired) → {grounded, movement}`. 2 tests (capsule lands+grounded; slides along wall). 195 tests green.

**Iter 2 DONE (code; browser pending):** player wired to the kinematic capsule. New ECS `Velocity {x,y,z,grounded}`; `RigidBody` += `collider` handle. `setup-character` builds the kinematic capsule (radius=planar half-extent ~0.3, halfHeight=extents[2]−radius ~0.6 → total ~1.8) + controller; sets `RigidBody.handle/collider`; adds `Velocity`. `CharacterControllerSystem` rewritten (ctor `(world, physics, keyboard, config, controller, camera)`): horizontal = camera-relative target (instant), `vz` = reset-on-ground + jump + gravity, → `physics.moveCharacter` → writes `Velocity`+`grounded`. `CharacterAnimationSystem` reads ECS `Velocity`/`grounded` (ctor `(animController, playerEid, character, config)`). `PhysicsSystem` unchanged. Note: in tests, call `physics.step()` so Rapier's query pipeline exists or `computeColliderMovement` sees nothing. 195 tests green. **Browser ✅ — slides through/around obstacles, never stuck.**

**Iter 3 DONE (code; browser pending):** run inertia. `MovementConfig` += `accel`/`deceleration`/`airControl`. Controller `approach`es horizontal velocity toward the input target by `(moving?accel:deceleration)·(grounded?1:airControl)·step` (one vector lerp-toward = ramp-up + decel + turn momentum); vertical unchanged so momentum carries into jumps (reduced air control). Defaults: `accel 20, deceleration 25, airControl 0.3` (walk 3 / run 7 / jump 3.5). Controller tests: ramps-not-instant, reaches target after sustained input, decelerates on release. 196 tests green.

**Iter 4 DONE — PLAN 013 COMPLETE (browser ✅).** Polish: (1) **foot-slide fix** — `AnimationController.setSpeed` scales walk/run clip playback to actual ground speed (`speed/authored`; authored walk 1.52 / run 4.31 u/s from IFP root motion; clamp 0.6–2.2). (2) **smooth turn** — `CharacterAnimationSystem` eases `facing` toward movement dir at `TURN_SPEED` 12 rad/s (no snap when camera rotated). (3) **procedural bounce** — small vertical bob on the inner model (`applyBounce`), phase by distance (`BOB_FREQUENCY 7` ≈2/stride), `BOB_AMPLITUDE 0.007` ramped to walk speed and held (run doesn't amplify — fixed run shake). Final speeds: `walkSpeed 2, runSpeed 7, jumpSpeed 3.5, accel 20, deceleration 25, airControl 0.3`. Player = kinematic capsule + Rapier character controller: slides around obstacles, autostep, slopes+snap, run inertia, speed-matched feet, smooth turn, gentle bounce. All tunables in `Config.movement`.

**Iterations:** 1) PhysicsWorld plumbing (createCharacterController, createKinematicCapsule Z-aligned, moveCharacter) — no wire-in; 2) wire player to kinematic capsule (setup-character + controller + animation reads ECS Velocity) — browser: slides round poles, autostep kerbs/stairs, slopes+snap; 3) run inertia (accel/decel/momentum/airControl, momentum into jumps); 4) tune (capsule/step/slope/accel).
