The map-loading layer renders a GTA San Andreas scene from its data files, built on top of the [[renderware-loader]]. All of it now lives under `src/renderware/` (post engine refactor — see [[engine-refactor-status]]); the engine consumes it only through `game/adapters/gta-sa-world.adapter.ts`.

**Parsers** — `src/renderware/parsers/text/` (pure, no three.js): `gta-dat` (`parseGtaDat` → `{ img, ide, ipl }` directive paths), `ide` (`parseIde` → object defs `id/model/txd/drawDistance/flags` from `objs`), `ipl` (`parseIpl` → text `inst` instances), `ipl-binary` (`parseBinaryIpl` → binary `bnry` stream IPLs: header magic `bnry`, `numInst`@0x04, `instOffset`@0x1C, 40-byte INST = pos 3f / quat 4f / **modelId** u32 / interior i32 / lod i32; `modelName=''`, resolved by id via catalog), `lod` (`isLodModel` — SA LOD models are `lod`-prefixed). `text-lines` has shared `sectionedParse`/`cleanLines`/`splitRow`. (RW binary parsers — dff/txd/chunks — are in `parsers/binary/`.)

**Resolution + region build** — `src/renderware/map/`: `resolve-map.ts` (`resolveMap(datUrl, base)` — async, framework-agnostic: fetch+parse dat→IDEs→catalog, text IPLs **plus** binary streams; was the old `useGtaMap`). Binary streams are driven by a **manifest** `static/ipl_binary/manifest.json` (`{ basename: streamCount }`, via `scripts/gen-ipl-manifest.mjs` / `npm run gen:ipl-manifest`) — loads exactly `ipl_binary/<base>_stream{0..count-1}.ipl`, **no probe-by-404** (`MAP.ZON`→`map_stream0.ipl` used to 404); `.ZON` directives are skipped (zones, not placement). `build-region.ts` (`buildRegion(archive, defs, { center, geometry, radius })` → `InstancedMesh[]`): filters to `interior === 0`, the chosen geometry kind (LODs vs map via `isLodModel`), and a radius around `center`; groups by model+txd key; one InstancedMesh per single-material part; matrices `compose(pos, quat.conjugate(), 1) × part.matrix` (**SA IPL quats are inverted → conjugate**); stamps `userData.region = { def, instances }` for click-inspect. Archive access via `getClump`/`getTextures`/`modelKey` (`renderware/archive`).

**Key design:** models stay native Z-up; the adapter wraps the built meshes in one group with a −90°X rotation (Z-up→Y-up). Camera bbox-framing after each region load lives in `game/core/camera-controller.ts` (GTA world coords are far from origin). The R3F walker (`use-gta-map`/`map-instance`/`map-scene`/`fit-camera`, `DFFLoader`/`TXDLoader`, `convertToYUp` flag) is **gone** — replaced by the above.

**Assets:** served from `static/` (`serve:static`, `VITE_STATIC_URL`). Real GTA SA Los Santos set: `static/data/gta.dat` (9 LA IDEs + 9 IPLs), `static/data/maps/LA/*.{ide,ipl}`, loose models in `static/img/gta3/` (~14.6k dff/txd, gta3.img extracted) + `static/img/gta3additional/` (props missing from the dump — gym/tray; `pack-img.mjs` packs both folders), binary stream IPLs in `static/ipl_binary/` (`<region>_stream<N>.ipl`), plus the packed WIMG archive `static/models/gta3.img` (single-download, in-memory unpack via `renderware/archive`). Catalog ~2514 defs; binary streams add ~10k INST of which ~1.2k resolve (def+file) → ~1,200 full-detail objects render (vs ~20 from text IPLs alone). Plans: `.claude/plans/002-gta-map-parsers.md`, `003-binary-ipl-streams.md`. Extension points (out of scope): generic-IDE coverage for the ~9k unresolved props/vegetation, geometry-merge perf pass (~1.2k draw calls), `.col` collision, binary IDE, interiors, streaming/LOD (phase 6 of plan 006).
