Plan: `.claude/plans/010-collision-streaming.md`. Make static map collision **stream per grid cell** (the render grid from [[world-streaming-plan]]) around the player, replacing the fixed Ganton zone built once at spawn (plan 008 / [[character-physics-plan]]). Cells gain Rapier static bodies when near, drop them when far → collision works everywhere the player drives.

**Reuses:** `buildCollisionIndex` (name→ColModel), `buildColliders` (bind by name, conjugated-quat, Z-up), `toModelColliders` (adapter DTO), `WorldGrid` (already built in the adapter for render), `cellsWithin` grid math. Physics stays GTA Z-up.

**Design:** (1) factor `bindColliders` out of `buildColliders`; add `buildCellColliders(index, defs, grid, cx, cy)` (cell HD instances only — LODs have no collision); (2) `WorldAdapter.loadCellColliders(cx,cy)` cached; (3) `PhysicsWorld.createStaticColliders` returns **body handles** (`number[]`, was count) + new `removeBodies(handles)` (`world.removeRigidBody`); (4) `CollisionStreamingSystem` (game/streaming) — view cell → desired = HD cells within `collisionDrawDistance` → diff `Map<cellKey,handles>` → create/remove Rapier bodies, async load with a margin radius (colliders ready before the player arrives); (5) `Config.streaming.collisionDrawDistance` (~150, small + margin); (6) `setupCharacter` registers it instead of the one-shot `createStaticColliders`, remove the fixed `Game.loadColliders`/`CollisionWorld` path.

**Iterations:** 1 `buildCellColliders` → 2 PhysicsWorld handles+`removeBodies` → 3 adapter `loadCellColliders` → 4 `CollisionStreamingSystem` (diff) → 5 wire-in + remove fixed zone + browser acceptance (drive far, cube lands everywhere). Decisions: separate system (not extend render StreamingSystem); separate small `collisionDrawDistance`; run on `update` with margin; Show Collision stays region-based for now; remove `CollisionWorld`/`loadColliders`.
