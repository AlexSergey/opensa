# 006 — procobj → static IPL (place scattered trees as fixed instances)

`procobj.dat` scatter is generated by `CPlantMgr` at runtime around the camera, so it has **no fixed positions**
and can't carry instance LODs. This stage converts the `--dff ∩ procobj` species into **static IPL
instances** (reusing the engine's own scatter), removes them from `procobj.dat`, and writes a dedicated
`lodtrees_procobj.ipl` — so they become fixed objects that get an impostor LOD like the placed trees.

## The core is NOT changed

It is tempting to "fix" the engine's `scatterProcObjects` to honour `procobj.dat`'s MINDIST column, but that
column is **the camera-creation radius in SA's `CProcObjectMan`, not a min-spacing** (per plan 042). The engine
renders procobj at full density per cell near the camera — correct, matching the game (≈639 per cell). The reason
the whole-map total is huge (~2 M) is simply that procobj is **never fully materialised** — it streams a capped
set within ~60 m of the camera. So the renderer stays as-is; **the thinning lives only in this export.**

## Scope (decided)

**Every** `--dff ∩ procobj` species is converted — the user curates the set through `--dff` (no model-type
heuristics in the tool). `--procobj-height` (default 0, off) is an optional gate to drop short clutter if wanted.
On the stock set this is ~51 species (trees, rocks, bushes, rubble, flowers, …), thinned by MINDIST + the cap.

## Thinning

Static can't materialise the full runtime density, so we reuse the engine's vanilla `scatterProcObjects`
(whole-map colliders), take the vanilla set (`lottery < 1`), then:

1. **MINDIST min-spacing** — a greedy spatial-hashed Poisson cull per species (the `procobj.dat` MINDIST as a
   real min-distance, which the renderer ignores) — drops dense clumps to a natural ~hundreds per species.
2. **Global cap** `--procobj-max` (default 20 000) — lowest-lottery survivors, a safety ceiling.

Scale is uniform `1.0` (SA's IPL `inst` has no scale field, so `MINSCL/MAXSCL` size variation is lost — accepted;
trees aren't `align`ed so only the yaw quaternion is written).

## Output (under `--out`)

- `data/maps/lodtrees_procobj.ipl` — per object an HD instance (stock model id, `lod` → its impostor) + the
  impostor LOD instance (text-internal link). HD models are stock; impostors are already in `lodtrees.ide`.
- `data/procobj.dat` — the converted species' rows removed (they no longer scatter at runtime).
- `data/gta.dat` — `+ IPL DATA\MAPS\lodtrees_procobj.IPL`. The impostor DFFs/TXD/COL are already in `gta3.img`.

## Module shape

`adapters/gta-sa/procobj/`: `world.ts` (assemble `MapDefinitions` — catalog + instances — for the engine
collision/scatter), `convert.ts` (gate + scatter reuse + MINDIST cull + cap + IPL/procobj/gta.dat emit). Imports
`buildColliders`/`buildCollisionIndex`/`scatterProcObjects`/`parseProcObj`/`parseSurfaceNames` from the engine.

## Caveats

- Each converted object is **2 instances** (HD + impostor LOD); at the default cap that's ~20 k objects ≈ 40 k
  instances on top of the ~10 k placed-tree LODs → watch SA's `CPool<CBuilding>` (a limit adjuster may be
  needed). Lower `--procobj-max` to trim.
- Converting a dense ground-cover species (grass/bushes) removes it from the runtime scatter and replaces it with
  the thinned static set — that ground gets sparser. Drop such species from `--dff` (or raise `--procobj-height`)
  to keep them on the runtime scatter.
